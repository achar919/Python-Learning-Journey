# History and Design of Python

## Overview

Python was created to solve problems that earlier programming languages struggled with, such as complex syntax, poor readability, manual memory management, and excessive boilerplate code. Its primary focus is simplicity, readability, and developer productivity.

---

# Evolution of Programming Languages

Programming languages evolved because developers continuously searched for better ways to solve problems.

| Language     | Year  | Main Purpose               | Limitation                          |
| ------------ | ----- | -------------------------- | ----------------------------------- |
| Machine Code | 1940s | Direct hardware control    | Difficult to write and maintain     |
| C            | 1972  | Systems programming        | Manual memory management            |
| Shell        | 1970s | System automation          | Not suitable for large applications |
| Perl         | 1987  | Text processing            | Difficult to maintain               |
| Java         | 1995  | Platform independence      | Verbose syntax                      |
| Python       | 1991  | Simplicity and readability | Slower execution speed              |

---

# Why Python Was Created

Python was created by **Guido van Rossum** in 1991 at CWI (Netherlands).

The goal was to create a language that:

* Is easy to read
* Is easy to learn
* Reduces unnecessary complexity
* Allows developers to focus on solving problems
* Supports both beginners and experienced developers

Python was heavily inspired by the ABC programming language.

---

# Python Design Philosophy

## 1. Readability

Python code should be easy to read and understand.

### Example

```python
name = input()
print(name)
```

Python avoids unnecessary complexity and encourages clear code.

---

## 2. Simplicity

Python removes many low-level concerns from the developer.

Examples:

* No manual memory allocation
* No memory deallocation
* Less boilerplate code
* Cleaner syntax

Developers focus on solving problems rather than managing the language.

---

## 3. Indentation

Python uses indentation to define code blocks.

### Example

```python
age = 20

if age >= 18:
    print("Adult")
```

Unlike many languages, Python does not require curly braces `{}`.

Indentation improves readability and enforces consistent code structure.

# Compiled vs Interpreted Languages

Computers understand only machine code. Programming languages must be translated before execution.

---

## Compiled Languages

Examples:

* C
* C++
* Java

Process:

```text
Source Code
    ↓
 Compiler
    ↓
 Executable
    ↓
 Execution
```

### Advantages

* Faster execution
* Optimized performance

### Disadvantages

* Compilation required after changes

---

## Interpreted Languages

Examples:

* Python

Process:

```text
Source Code
    ↓
 Interpreter
    ↓
 Execution
```

### Advantages

* Faster development cycle
* Easier testing and experimentation

### Disadvantages

* Slower execution speed
